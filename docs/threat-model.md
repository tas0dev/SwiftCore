# Threat Model

## 保護対象 (Assets)

- カーネル `.text` / 制御フロー整合性
- ページテーブルとアドレス空間分離
- プロセス/スレッド境界
- サービス管理プロセスの一意性
- カーネル内機密情報 (キー、内部ポインタ、状態)

## 想定攻撃者

- 悪意あるユーザープロセス
- 破損したサービスバイナリ
- 入力値で境界を崩すクラッシュ誘発者

## 信頼境界

- ユーザー空間 ↔ カーネル空間 (syscall/interrupt)
- コアサービス ↔ カーネル API
- ブートローダ由来情報 ↔ 実行時メモリ管理

## 主要攻撃面

1. **Syscall 引数**
   - 無効ポインタ
   - 長さ偽装
   - 競合による TOCTOU
2. **メモリ管理**
   - CR3 切り替えミス
   - ページ属性設定ミス
3. **スレッド/プロセス管理**
   - wait/fork のハング
   - 状態遷移競合
4. **IPC**
   - 宛先消滅時の整合性
   - 送受信境界での検証不足

## 非目標 (現時点)

- 物理攻撃耐性
- サイドチャネル完全防御
- 完全 SMP 最適化

## 既存対策の例

- `.text` 領域の Read-Only 化
- NXE / SMEP / SMAP 有効化
- KPTI の CR3 切り替え
- `validate_user_ptr` + `with_user_memory_access`
- SYSRET 前の non-canonical RSP 検証 (CVE-2012-0217 緩和)

## 継続課題

- futex の厳密な待機/起床モデル
- syscall/int 経路のコンテキスト保存統一
- ユーザーメモリアクセスの原子性改善
