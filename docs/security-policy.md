# Security Policy

## 対象

本ポリシーは `dev` ブランチと、そこから派生する開発ブランチを対象とします。

## 脆弱性報告

- 公開 Issue でのゼロデイ詳細公開は避けてください。
- 再現手順、影響範囲、想定される悪用経路を含めて報告してください。
- 重大度は以下で分類します:
  - **Critical**: 権限昇格、任意コード実行、カーネルメモリ破壊
  - **High**: サンドボックス逸脱、情報漏えい、永続 DoS
  - **Medium**: ローカル DoS、境界チェック欠落、競合窓
  - **Low**: 防御深度の不足、将来リスク

## 修正方針

1. Critical / High を優先して先に修正する
2. 回帰テスト (build/test/fmt) を必須とする
3. 仕様変更が入る場合はドキュメントを同時更新する
4. 一時回避ではなく根本原因を優先して修正する

## マージ条件

セキュリティ修正 PR は以下を満たす必要があります:

- CI 成功 (`fmt`, `build`, `test`)
- 影響範囲の説明
- 回帰リスクの説明
- 必要なドキュメント更新

## 現在の重点領域

- SYSCALL 境界のユーザーコンテキスト保存
- ユーザーポインタ境界検証
- CR3 切り替え防御
- IPC / futex の競合窓の明文化と段階的改善
