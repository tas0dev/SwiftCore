# Secure Coding Guidelines

この文書は SwiftCore にコードを追加・変更する際の必須ルールです。

## 1. Syscall 境界

- ユーザー入力は **必ず** 検証する:
  - ポインタ: `validate_user_ptr`
  - サイズ: 上限チェック + overflow チェック (`checked_add`)
- ユーザーポインタの実アクセスは `with_user_memory_access` 配下で行う。
- チェックと使用が分離される場合は TOCTOU 窓をコメントで明記する。

## 2. unsafe 使用

- unsafe は最小スコープに限定する。
- `from_raw_parts` / `from_raw_parts_mut` は長さと境界を先に保証する。
- 生ポインタループでは「読み出し境界」と「書き込み境界」を別々に検証する。

## 3. 権限とメモリ

- ユーザー指定アドレスを CR3/FS/GS 等の制御レジスタへ反映する前に検証する。
- ページテーブルアドレスは 4KiB アラインを必須とする。
- カーネル `.text` に対する writable マッピングは許可しない。

## 4. 同期と競合

- グローバル共有状態に syscall 入口コンテキストを直接保存しない。
- スレッド固有データはスレッド構造体または per-CPU 領域で保持する。
- ロック保持時間を短くし、不要なロック取得を避ける。

## 5. 失敗時の振る舞い

- サイレント失敗を避け、適切な errno を返す。
- 無限待機ループには timeout か終了条件を設ける。
- エラー時の資源解放を必ず行う。

## 6. レビュー時チェック項目

- [ ] ユーザー入力境界が全経路で検証されている
- [ ] 競合窓が必要十分に制御または文書化されている
- [ ] 権限境界を跨ぐ書き込みが検証されている
- [ ] 例外経路でリーク/ハングがない
- [ ] CI (`fmt/build/test`) が通る
